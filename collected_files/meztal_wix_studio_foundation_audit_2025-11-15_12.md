## RISKS IF FOUNDATION NOT LAID PROPERLY

### Risk 1: Orphaned Pages
If keyword landing pages created without topical hub linking, they'll have no semantic context and poor ranking potential. **Mitigation:** Complete Phase 0 internal linking topology first.

### Risk 2: Duplicate Content
If FAQ and landing pages cover same keywords, search engines won't know which to rank. **Mitigation:** FAQ links TO pages, not vice versa; deduplicate content.

### Risk 3: Poor Schema Implementation
If schema markup added after pages exist, may need complete rebuild. **Mitigation:** Implement during page creation, not after.

### Risk 4: Scalability Failure
If all content hardcoded (not using collections), future case studies/testimonials require new pages. **Mitigation:** Set up collections in Phase 0, test with dummy data.

### Risk 5: Conversion Leakage
If all pages use same CTA ("Apply Now"), awareness-stage traffic converts poorly. **Mitigation:** Implement stage-based CTAs in Phase 1.
